# 데이터 보존·공개 기준

## 목적과 현재 상태

이 저장소는 약국 상품의 이름, 규격, 분류와 특정 시점의 표시 가격을 검색 가능한 연구 자료로 정리한다. 상품 판매, 구매 중개, 재고 안내 또는 의약품 광고를 목적으로 하지 않는다.

현재 데이터는 **로컬 연구 아카이브**다. 외부 공개용 데이터로 승인된 상태가 아니다. 로컬 아카이브에 저장한 정보가 많다는 이유만으로 모든 정보를 외부에 공개하지 않는다.

## 로컬 아카이브에는 가격을 포함한 원자료를 보존한다

- 공개 Firestore `products` 컬렉션과 화면 녹화에서 확인한 상품명, 규격, 분류와 표시 가격을 삭제하지 않는다.
- Firestore 정본에는 원본 문서 ID, 생성·수정 시각과 조회일을 저장한다. 화면 녹화 OCR 대조본에는 출처 파일, 화면 위치, 문자 인식 신뢰도와 검수 상태를 저장한다.
- 가격은 `2026-07-15 조회 당시 앱 데이터값`으로 저장한다. 현재 판매가, 현재 재고 또는 구매 가능 여부로 바꾸어 쓰지 않는다. 화면 녹화 OCR 값은 정본이 아니라 대조 자료로만 사용한다.
- OCR 원문과 검수 전 값은 비공개 작업 데이터로 보존한다. Git 저장소와 외부 배포 파일에는 넣지 않는다.
- 녹화 중 표시된 메신저 알림, 사람 이름, 계정 정보와 다른 개인정보는 상품 자료가 아니므로 제거한다.

## 데이터 파일의 공개 범위

- `data/firestore-products.json`: Firestore 원본 필드와 타입을 보존한 비공개 로컬 정본이다. Git과 외부 배포에 포함하지 않는다.
- `data/products.raw.json`: OCR로 추출한 비공개 대조 데이터다. Git과 외부 배포에 포함하지 않는다.
- `data/products.json`, `data/catalog.csv`: 가격과 출처 필드를 포함한 로컬 연구용 정리본이다. Git과 외부 배포에 포함하지 않는다.
- `public/data/products.json`, `public/data/catalog.csv`: 로컬 화면 확인을 위한 생성 파일이다. 파일명이 `public`이어도 외부 공개 승인을 뜻하지 않는다. Git과 외부 배포에 포함하지 않는다.
- 원본 앱 화면, 앱 로고, 광고와 앱이 촬영한 재고 사진은 외부 공개 파일에 포함하지 않는다.
- 식약처 등 독립된 공식 출처에서 확인한 정보는 화면 녹화에서 얻은 값과 별도 필드로 구분한다.

## 가격을 표시할 때 지킬 기준

- 모든 가격 옆에 조회일과 `조회 당시 앱 데이터값` 상태를 표시한다.
- 목록 상단과 상품 상세 화면에 `현재 판매가·재고 아님` 안내를 눈에 띄는 크기로 표시한다.
- OCR 검수 전 가격은 `검수 필요`로 표시하고 정렬·통계에 포함할 때도 검수 상태를 함께 제공한다.
- 가격 비교, 할인율, 최저가, 구매 권유처럼 현재 거래 조건으로 오인할 표현을 만들지 않는다.
- 가격의 현재성을 확인하려면 이용자가 방문할 약국에 직접 문의하도록 안내한다.

## 이미지 사용 기준

- 식약처 공공데이터처럼 이용 조건이 확인된 이미지만 자동 표시한다.
- 식약처 e약은요의 낱알이미지를 사용하면 공공데이터 원본 URL, 제공기관, 확인일과 당시 표시된 이용허락범위를 함께 저장한다.
- 제조사 공식 페이지의 이미지는 별도 이용 허락이나 명확한 재사용 조건을 확인한 뒤 추가한다. 공식 페이지에 공개되어 있다는 사실만으로 재사용 허락을 추정하지 않는다.
- 쇼핑몰, 블로그와 검색 결과 이미지는 파일로 복사하지 않는다. 상품명이 충분히 일치하는 경우에만 출처 페이지와 함께 원격 미리보기로 표시한다.
- 짧거나 모호한 상품명, 규격·제형 충돌 또는 다른 상품 가능성이 있는 후보는 자동 표시하지 않고 `review_required`로 보낸다.

## 외부 공개 게이트

다음 조건을 모두 충족하기 전에는 전체 상품 JSON·CSV, 전체 가격표와 원본 화면을 외부에 배포하거나 공개 저장소에 올리지 않는다.

1. **취득 경위 확인:** 앱 이용약관, 접근 권한과 수집 과정에 기술적 보호조치 우회가 없었는지 기록한다.
2. **데이터베이스 권리 검토:** 전체 또는 상당한 부분의 공개와 반복적·체계적 추출이 저작권법상 데이터베이스제작자의 권리와 충돌하는지 검토한다.
3. **연구 목적 예외 검토:** 저작권법 제94조의 비영리 교육·학술·연구 목적 요건과 데이터베이스의 일반적인 이용에 저촉되지 않는다는 요건을 구체적으로 확인한다. `연구용`이라는 표지만으로 예외가 자동 적용된다고 보지 않는다.
4. **부정경쟁 위험 검토:** 부정한 데이터 취득·사용 또는 타인의 상당한 투자·노력으로 만든 성과를 공정한 거래질서에 반해 영업에 무단 사용한다는 평가를 받을 가능성을 검토한다.
5. **독립 출처 대조:** 식약처·제조사 등 독립된 출처에서 재확인한 필드와 앱 화면에서만 확인한 필드를 구분한다.
6. **이미지 권리 확인:** 이미지마다 재사용 근거, 원본 URL, 제공기관과 확인일을 기록한다. 근거가 없는 이미지는 제외한다.
7. **가격 오인 방지 확인:** 기록일, `현재 판매가·재고 아님`, OCR 검수 상태가 가격과 가까운 위치에 충분히 큰 글자로 보이는지 실제 화면에서 확인한다.
8. **정정·삭제 절차 마련:** 실제 운영 연락처, 접수 기록, 임시 비공개와 처리 기준을 마련한다.
9. **독립성 표시:** 메가팩토리약국 또는 앱 운영자의 공식 서비스, 제휴 서비스나 승인받은 가격표로 보이지 않는 이름·로고·색상·화면 구성을 사용한다.
10. **배포 승인 기록:** 법률 검토 결과, 공개 필드, 공개 근거, 승인자와 승인일을 저장한다.

## 법적 근거를 읽을 때 유의할 점

저작권법 제93조는 데이터베이스 구성 소재 자체와 데이터베이스 제작자의 권리를 구분하지만, 데이터베이스의 전부·상당한 부분 또는 반복적·체계적인 일부 이용에는 별도 권리가 문제될 수 있다고 규정한다. 제94조에는 비영리 교육·학술·연구 목적의 제한 규정이 있지만, 데이터베이스의 일반적인 이용과 저촉되지 않아야 한다.

부정경쟁방지법 제2조제1호 카목은 일정한 데이터의 부정한 취득·사용 등을, 파목은 타인의 상당한 투자나 노력으로 만든 성과를 공정한 상거래 관행이나 경쟁질서에 반하는 방법으로 영업에 무단 사용해 경제적 이익을 침해하는 행위를 규정한다. 구체적 적용 여부는 취득 방법, 이용 목적, 공개 범위, 경쟁·대체 관계와 이용약관 등을 함께 검토해야 한다.

표시·광고의 공정화에 관한 법률은 소비자가 잘못 알 우려가 있는 거짓·과장 또는 기만적인 표시·광고를 금지한다. 이 아카이브가 상품을 판매하지 않더라도, 과거 가격을 현재 거래 가격처럼 보이게 만들지 않는 기준을 적용한다.

이 문서는 법률 자문이 아니다. 전체 데이터의 외부 공개, 공개 API 제공 또는 상업적 이용 전에는 대한민국 변호사의 검토가 필요하다.

## 공식 공개 출처 보강 절차

공식 정보 보강은 `data/products.json`에 바로 쓰지 않고 별도 조사 큐에서 처리한다. 조사 순서는 다음과 같다.

1. 식약처 의약품 제품 허가정보
2. 식약처 의약품개요정보(e약은요)
3. 식약처 건강기능식품정보
4. 한국식품안전관리인증원 HACCP 제품이미지 및 포장지표기정보
5. 식약처 기능성화장품 보고품목정보
6. 식약처 의료기기 품목허가정보

`scripts/enrich_public_sources.py`는 공공데이터포털 인증키를 환경 변수 `DATA_GO_KR_SERVICE_KEY`에서만 읽는다. 인증키를 파일, 실행 결과 또는 오류 로그에 저장하지 않는다. 인증키가 없으면 기존 조사 큐와 결과 파일을 바꾸지 않고 누락 요약 JSON만 생성한다.

공식 제품 연결은 다음 기준을 적용한다.

- 공식 품목코드·바코드·UDI-DI가 같으면 자동 확정한다.
- 정규화 상품명이 정확히 같고 3점 이내의 경쟁 후보나 명시적인 제형·함량 충돌이 없으면 자동 확정 후보로 분류한다. 판매 포장 규격은 공식 허가 포장단위와 다를 수 있으므로 규격만으로 확정하거나 제외하지 않는다.
- 종합 점수가 95점 이상이어도 3점 이내의 경쟁 후보가 있으면 `review_required`로 보낸다.
- 종합 점수가 80~94점이면 사람이 제품명, 규격, 제조사와 공식 품목코드를 확인한다. 80점 미만 후보는 제품 정보에 합치지 않는다.
- 중복 상품 그룹은 같은 공식 품목코드로 자동 병합하지 않는다. 규격과 제조사가 다르면 별도 상품으로 유지한다.
- 공식 출처에 없는 브랜드명은 상품명에서 추정해 확정하지 않는다.

공식 API의 개방 데이터와 제조사 웹사이트의 콘텐츠는 구분한다. 공공데이터포털에서 이용허락범위가 확인된 데이터만 그 범위 안에서 사용한다. 제조사 공식 페이지는 출처 URL 후보로 저장할 수 있지만, 공개된 페이지라는 이유만으로 이미지 복제 허락이 있다고 보지 않는다.

e약은요의 `itemImage`는 제품 포장 이미지가 아니라 낱알 이미지이므로 `image_kind=pill`로 기록한다. HACCP 제품이미지는 `image_kind=package`로 기록한다. 이미지마다 개별 공식 레코드 URL, 제공기관, 이용허락범위와 확인 시각을 함께 저장한다. 제품 연결이 `review_required` 상태이면 이미지 URL을 최종 제품 필드에 자동 반영하지 않는다.

## 약학정보원 자료의 출처와 공개 위험을 별도로 관리함

약학정보원 서비스 이용약관은 서비스에서 얻은 정보를 사전승낙 없이 이용자 개인의 이용 외 목적으로 복제하거나 제3자에게 제공하는 행위를 제한한다. 현재 연구본은 제품명 일치 여부를 검증한 뒤 성분·효능·용법·주의사항 등을 별도 필드로 저장하고, 각 제품에 약학정보원 원문 링크를 연결한다. 이 구현은 약학정보원의 재사용 승인이나 제휴를 뜻하지 않는다.

- 이용약관: https://www.health.kr/member/userAgreement.asp
- 법적책임의 한계: https://www.health.kr/member/legalLiability.asp
- 콘텐츠 제휴 문의: https://www.health.kr/notice/inquireAd.asp

포장·낱알 이미지는 로컬 파일로 복제하지 않고 약학정보원 원문 서버의 미리보기 주소와 출처 URL만 저장한다. 공개 운영이나 상업적 이용을 계속하려면 약학정보원에 필드·건수·이용 목적·공개 범위·저장 기간·갱신 주기를 제시하고 서면 승인을 확인해야 한다. 권리자가 정정·삭제를 요청하면 확인이 끝날 때까지 해당 본문과 이미지를 비공개 대상으로 처리한다.

공식 제품 상세 수집기는 공공데이터포털에서 이용허락범위가 `제한 없음`으로 표시된 데이터셋 `15095677`, `15075057`, `15057639`, `15095679`, `15056760`, `15095680`, `15056939`, `15073875`, `15033307`을 사용한다. 각 값에는 데이터셋 ID, 공식 레코드 ID, 출처 URL, 수집 시각, 상위 데이터 수정일, 원문 해시와 이용허락 문구를 저장한다.
