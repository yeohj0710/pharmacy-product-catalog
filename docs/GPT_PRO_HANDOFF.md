# GPT Pro 공식 제품 매칭 작업 지침

## 작업 단위

`data/gpt-pro-official-review-queue.csv`에서 `batch_no` 하나만 골라 최대 25개씩 조사한다. 입력 행과 순서를 삭제하거나 바꾸지 않는다.

## 허용 출처

다음 공공데이터포털 데이터셋과 식품의약품안전처 공식 페이지를 우선 사용한다.

- 의약품 제품 허가정보 `15095677`
- 의약품개요정보(e약은요) `15075057`
- 의약품 낱알식별정보 `15057639`
- 의약외품 제품 허가정보 `15095679`
- 건강기능식품정보 `15056760`
- 기능성화장품 정보 `15095680`, `15056939`
- 의료기기 표준코드별 제품정보 `15073875`
- HACCP 제품이미지 `15033307`

약학정보원 화면의 본문과 이미지는 복사하지 않는다. 약학정보원은 후보를 눈으로 확인하는 링크로만 사용할 수 있으며, `official_source_url`에는 실제 값을 가져온 공식 공공데이터 출처를 적는다.

## 확정 기준

1. 품목기준코드·품목제조관리번호·UDI-DI 또는 바코드가 같으면 확정 후보로 기록한다.
2. 식별자가 없으면 정규화 제품명이 정확히 같고 후보가 하나이며 제형·함량 충돌이 없어야 확정한다.
3. 제품명 점수가 80~94이거나 3점 이내의 경쟁 후보가 있으면 `review_required`로 기록한다.
4. 건강기능식품·의료기기·화장품은 약학정보원에 없다는 이유로 `not_found` 처리하지 않는다. 제품 영역에 맞는 공식 API를 확인한다.
5. 판매 규격 `10포`, `20정`은 성분 함량 `500mg`과 다른 값이다. 두 값을 같은 매칭 키로 사용하지 않는다.

## 출력 규칙

- CSV 열 이름을 변경하지 않는다.
- 긴 텍스트는 원문 의미를 바꾸지 않고 그대로 기록한다.
- 성분과 이미지는 JSON 배열 문자열로 기록한다.
- 이미지에는 `kind`, `url`, `source_url`, `license`가 모두 있어야 한다.
- 모델 지식으로 효능, 용법, 주의사항이나 성분을 작성하지 않는다. 공식 원문에 없는 값은 빈칸으로 둔다.
- `match_reason`에는 일치한 식별자와 제품명·업체·제형의 비교 결과를 한 문장으로 적는다.
- CSV 행은 `schemas/gpt-pro-official-review-row.schema.json`의 열과 상태값을 사용한다.
- 내부 매칭 레코드는 `schemas/official-match.schema.json`을 사용한다. CSV 열을 내부 JSON 필드로 임의 변경하지 않는다.
- `lookup_status=found`라도 후보가 모호하면 `match_status=review_required`로 기록한다.
